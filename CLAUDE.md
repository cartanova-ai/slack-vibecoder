# slack-vibecoder 프로젝트 지침

## 작업 원칙

- 간단한 프로젝트이니, 어지간한 것을 알아서 해 주십시오.
- 그렇지만 모든 결정 전에 사용자에게 충분히 물어보십시오. 알아서 하라는 것은 사용자가 좋아할 선택지들을 잘 만들어서 제시하라는 것입니다.
- 작업이 끝나면 커밋하고 푸시하는 것이 기본이지만, 혹시 당신이 빠뜨린 것이 있을 지도 모르니 푸시하기 전에 사용자에게 물어보십시오.

## 버전 관리 (SemVer)
slack-vibecoder 프로젝트에 변경사항을 커밋하고 푸시할 때는 반드시 package.json의 version 필드를 SemVer 규칙에 맞춰 업데이트해야 합니다.

- MAJOR.MINOR.PATCH 형식 (예: 1.0.0)
- MAJOR: 호환되지 않는 API 변경
- MINOR: 하위 호환성을 유지하면서 기능 추가
- PATCH: 하위 호환성을 유지하면서 버그 수정

버전 업데이트 예시:
- 버그 수정: 1.0.0 → 1.0.1
- 기능 추가: 1.0.1 → 1.1.0
- 주요 변경: 1.1.0 → 2.0.0
